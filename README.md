# 🏭 Система промышленного мониторинга воздуха

Полный стек прошивок для двухуровневой системы мониторинга качества воздуха:
- **Уровень 1** — STM32 считывает пыль (SPS30) и передаёт данные по Modbus RTU через RS-485
- **Уровень 2** — сеть ESP32 узлов передаёт данные по ESP-NOW на центральный шлюз без Wi-Fi роутера

---

## 📁 Структура проекта

```
├── STM32_SPS30_ModbusRTU.ino   # STM32: датчик SPS30 → Modbus RTU (RS-485)
├── ESP32_Node.ino              # ESP32 Узел: сбор данных → ESP-NOW передача
├── ESP32_Master.ino            # ESP32 Шлюз: приём данных от всех узлов
└── README.md
```

## ⚙️ Часть 1 — STM32 + SPS30 + Modbus RTU

### Описание

STM32 опрашивает датчик частиц Sensirion SPS30 по I2C и предоставляет данные как **Modbus RTU Slave** через интерфейс RS-485. Устройство рассчитано на непрерывную работу 24/7.

### Схема подключения

```
SPS30                STM32
─────────────────────────────
SDA         →        PB7 (I2C1 SDA)
SCL         →        PB6 (I2C1 SCL)
SEL         →        GND  (режим I2C)
VDD         →        5V
GND         →        GND

RS-485 модуль        STM32
─────────────────────────────
TX (DI)     →        PA2 (USART2 TX)
RX (RO)     →        PA3 (USART2 RX)
DE + RE     →        PA1 (управление направлением)
```

### Зависимости

| Библиотека | Версия | Источник |
|---|---|---|
| Sensirion I2C SPS30 | v1.x | Arduino Library Manager / PlatformIO |
| Wire | встроенная | Arduino Framework |
| IWatchdog *(опц.)* | встроенная | STM32duino |

### Карта Modbus регистров

Адрес устройства по умолчанию: **0x01**
Поддерживаемые функции: **FC03**, **FC04** (Read Holding Registers)
Формат значений: `float × 10` → `uint16` (одна десятичная цифра, например 12.5 → 125)

| Адрес | Регистр | Единица | Описание |
|---|---|---|---|
| 40001 | PM1.0 × 10 | мкг/м³ | Концентрация частиц PM1.0 |
| 40002 | PM2.5 × 10 | мкг/м³ | Концентрация частиц PM2.5 |
| 40003 | PM4.0 × 10 | мкг/м³ | Концентрация частиц PM4.0 |
| 40004 | PM10 × 10 | мкг/м³ | Концентрация частиц PM10 |
| 40005 | NC0.5 × 10 | #/см³ | Числовая концентрация 0.5 мкм |
| 40006 | NC1.0 × 10 | #/см³ | Числовая концентрация 1.0 мкм |
| 40007 | NC2.5 × 10 | #/см³ | Числовая концентрация 2.5 мкм |
| 40008 | NC4.0 × 10 | #/см³ | Числовая концентрация 4.0 мкм |
| 40009 | NC10 × 10 | #/см³ | Числовая концентрация 10 мкм |
| 40010 | TPS × 10 | мкм | Типичный размер частиц |
| 40011 | Статус | — | 0 = OK, 1 = ошибка, 2 = прогрев |
| 40012 | Счётчик | — | Счётчик успешных измерений (0–65535) |

### Параметры конфигурации

Откройте `STM32_SPS30_ModbusRTU.ino` и измените константы в разделе `Конфигурация`:

```cpp
#define MODBUS_SLAVE_ID        0x01    // Адрес Modbus (1–247)
#define MODBUS_BAUD            9600    // Скорость RS-485
#define SPS30_MEASURE_INTERVAL_MS 1000 // Интервал опроса (мс)
#define SPS30_WARMUP_MS        30000   // Время прогрева SPS30 (мс)
```

### Включение аппаратного Watchdog (IWDG)

Добавьте флаг в `platformio.ini`:

```ini
[env:your_board]
build_flags = -DUSE_IWDG
```

Watchdog перезагрузит устройство, если основной цикл зависнет дольше **8 секунд**.

## ⚙️ Часть 2 — ESP32 Узел (Node)

### Описание

Каждый узел собирает данные с датчиков (PM2.5, SO2, NH3) и отправляет их на Master каждые 30 секунд по протоколу **ESP-NOW** — прямая радиосвязь между платами без роутера, дальность до ~200 м в прямой видимости.

### Зависимости

Все библиотеки входят в состав **ESP32 Arduino Core** — дополнительная установка не требуется.

### Быстрый старт

**Шаг 1.** Прошейте `ESP32_Master.ino` на плату шлюза и откройте Serial Monitor (115200 бод). Вы увидите строку:

```
[MASTER] Мой MAC: AA:BB:CC:DD:EE:FF
```

**Шаг 2.** Скопируйте MAC и вставьте его в `ESP32_Node.ino`:

```cpp
static uint8_t masterMacAddr[6] = {0xAA, 0xBB, 0xCC, 0xDD, 0xEE, 0xFF};
```

**Шаг 3.** Задайте уникальный ID для каждой платы узла:

```cpp
static const uint8_t NODE_ID = 1;  // 1, 2, 3 ... для каждого узла
```

**Шаг 4.** Прошейте `ESP32_Node.ino` на каждую плату узла.

### Схема подключения датчиков (пример)

```
Датчик SO2 (аналог)  →  GPIO34 (ADC, только вход)
Датчик NH3 (аналог)  →  GPIO35 (ADC, только вход)
SPS30 SDA            →  GPIO21 (I2C)
SPS30 SCL            →  GPIO22 (I2C)
```

> **Примечание:** Функции `readPM25()`, `readSO2()`, `readNH3()` содержат заглушки с тестовыми данными. Замените их вызовами реальных драйверов датчиков под ваше оборудование.

### Структура передаваемого пакета

```cpp
struct sensorData {
  uint8_t nodeId;   // ID узла (1 байт)
  float   pm25;     // PM2.5, мкг/м³ (4 байта IEEE 754)
  float   so2;      // SO2, ppm       (4 байта)
  float   nh3;      // NH3, ppm       (4 байта)
};
// Итого: 13 байт на пакет
```

### Механизм восстановления связи

```
Ошибка отправки
       │
       ▼
  failCount++
       │
  failCount ≥ 5?
    ├── НЕТ → продолжаем работу
    └── ДА  → реинициализация ESP-NOW
                   │
              Успех? ──НЕТ──→ ESP.restart()
                   │
                  ДА → failCount = 0, работа продолжается
```

## ⚙️ Часть 3 — ESP32 Шлюз (Master)

### Описание

Шлюз получает данные от всех узлов одновременно, выводит результаты в Serial Monitor, следит за доступностью узлов и формирует периодические сводки.

### Вывод в Serial Monitor

Каждый принятый пакет:
```
┌─────────────────────────────────────────
│ Датчик №1 зафиксировал:
│   PM2.5 = 14.3 мкг/м³
│   SO2   = 0.82 ppm
│   NH3   = 3.15 ppm
│   Пакетов получено: 47
└─────────────────────────────────────────
```

Сводка каждые 60 секунд:
```
╔══════════ СВОДКА СИСТЕМЫ ════════════╗
║  Активных узлов: 3/10
║  Uptime: 3720 с
║  Узел #1 [ONLINE ] PM2.5=14.3 SO2=0.82 NH3=3.15 (12.0 с назад)
║  Узел #2 [ONLINE ] PM2.5=22.1 SO2=1.10 NH3=0.90 (8.0 с назад)
║  Узел #3 [OFFLINE] PM2.5=0.0  SO2=0.00 NH3=0.00 (95.0 с назад)
╚══════════════════════════════════════╝
```

### Пороговые предупреждения

Встроенные лимиты (можно изменить в коде):

| Параметр | Порог | Сообщение |
|---|---|---|
| PM2.5 | > 35 мкг/м³ | `[!] ВНИМАНИЕ: PM2.5 превышает...` |
| SO2 | > 5 ppm | `[!] ВНИМАНИЕ: SO2 превышает...` |
| NH3 | > 25 ppm | `[!] ВНИМАНИЕ: NH3 превышает...` |

### Параметры конфигурации


## ❓ Часто задаваемые вопросы

**Q: Как добавить новый узел в работающую систему?**
A: Просто прошейте новую плату с уникальным `NODE_ID` и правильным MAC-адресом Master. Шлюз обнаружит узел автоматически при получении первого пакета.

**Q: Почему SPS30 не даёт показания сразу после включения?**
A: Датчик требует 30 секунд на раскрутку вентилятора. В это время регистр статуса (40011) показывает значение `2` (прогрев). После прогрева значение меняется на `0` (OK).

**Q: Какова максимальная дальность ESP-NOW?**
A: В прямой видимости — до 200–400 м. Через стены — 30–80 м в зависимости от материала. Для промышленных условий рекомендуется использовать ESP32 с внешней антенной.

**Q: Как изменить адрес Modbus устройства?**
A: Измените `#define MODBUS_SLAVE_ID 0x01` на нужный адрес (1–247) в файле `STM32_SPS30_ModbusRTU.ino` и перекомпилируйте.

**Q: Можно ли подключить несколько STM32 на одну шину RS-485?**
A: Да. Назначьте каждому устройству уникальный `MODBUS_SLAVE_ID`. Все они могут находиться на одной физической шине RS-485.
